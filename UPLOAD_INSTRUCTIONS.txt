üìã MANUAL METHOD: Step-by-Step
================================================================================

STEP 1: Create Repository on GitHub
------------------------------------
1. Go to: https://github.com/new
2. Repository name: modern-ecommerce
3. Description: Modern e-commerce website with glassmorphism design
4. Choose Public or Private
5. DO NOT check "Initialize with README"
6. Click "Create repository"
7. Copy the repository URL (looks like):
   https://github.com/YOUR_USERNAME/modern-ecommerce.git

STEP 2: Open Command Prompt
----------------------------
1. Press Win + R
2. Type: cmd
3. Press Enter

STEP 3: Navigate to Project
----------------------------
cd c:\xampp\htdocs\AISecurityTech\modern-ecommerce

STEP 4: Initialize Git (if not done)
-------------------------------------
git init

STEP 5: Configure Git (first time only)
----------------------------------------
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

STEP 6: Add All Files
----------------------
git add .

STEP 7: Create First Commit
----------------------------
git commit -m "Initial commit: Modern e-commerce website"

STEP 8: Connect to GitHub
--------------------------
git remote add origin https://github.com/YOUR_USERNAME/modern-ecommerce.git
(Replace YOUR_USERNAME with your actual GitHub username)

STEP 9: Push to GitHub
-----------------------
git branch -M main
git push -u origin main

STEP 10: Enter Credentials
---------------------------
When prompted:
- Username: Your GitHub username
- Password: Your Personal Access Token (NOT your password)

To create a Personal Access Token:
1. Go to: https://github.com/settings/tokens
2. Click "Generate new token (classic)"
3. Select "repo" scope
4. Click "Generate token"
5. Copy the token and use it as password

üêõ TROUBLESHOOTING
================================================================================

Problem: "git: command not found"
Solution: Install Git from https://git-scm.com/download/win

Problem: "Permission denied"
Solution: Use Personal Access Token instead of password

Problem: "Repository already exists"
Solution: Use different name or delete existing repository

Problem: "Failed to push"
Solution: 
  git pull origin main --rebase
  git push origin main

Problem: "Large files warning"
Solution: Make sure node_modules is in .gitignore
